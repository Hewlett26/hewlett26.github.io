FILE_001 — ORIGIN

How It Started

I started coding just over a year ago, and it all began with Linux. I had an old machine that couldn't run Windows properly, so I installed Arch Linux on a whim. That decision changed everything.

The terminal became my playground. I started writing small Bash scripts to automate things, then moved on to Python. Every problem I encountered became an excuse to build something. My first real project was Magnet — a package management wrapper that uses Distrobox to containerize Fedora and Debian on Arch, just because I wanted more software available.

What I realized early on is that building things that solve real problems — even small ones — is incredibly satisfying. And if they can look good while doing it, even better.

FILE_002 — PHILOSOPHY

What I Believe

I believe anyone can code. The barrier isn't intelligence or talent — it's finding the right way to translate what's in your head into something a computer can understand.

A website like this one isn't magic. It's just HTML, CSS, and a little JavaScript. Anyone with enough curiosity and patience can build it. The hard part isn't the code — it's having a clear vision of what you want to make and the discipline to see it through.

That's why I share my projects openly. If something I built helps one person realize they can build too, that's a win.

FILE_003 — APPROACH

How I Build

I learn by building. Watching tutorials and reading docs only gets you so far — the real learning happens when you're stuck on something and have to figure it out yourself.

My process is simple: have an idea, start building, get stuck, unstick myself, repeat. Every project teaches me something I didn't expect. Brew Haven taught me about layout and responsive design. Iron Vale taught me about structuring content. Magnet taught me about Linux internals and package management.

I want my projects to serve as inspiration for others. Not because they're perfect — they're not — but because they show what's possible when you just start building.

FILE_004 — VISION

Where I'm Headed

I want to build tools that make things easier for people. Software that doesn't get in your way. Websites that feel like they were made by a human, not a template.

I'm tired of the generic, cookie-cutter web that dominates everything today. There used to be niches — weird, creative corners of the internet where people made things because they loved making them. I want to help bring some of that back.

Whether it's through retro aesthetics, thoughtful interactions, or just building things that work the way they should, I want my work to feel intentional. Because the best technology is the kind you don't notice — it just works, and it feels right.
